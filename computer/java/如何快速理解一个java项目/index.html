如何快速理解一个java项目？

# 了解需求，对应架构（不要去看代码，自己思考怎么做）

需求：

​ 从项目开发人员或客户，明白这个项目的需求是什么（即：这个项目整体需要实现怎样的功能）；

架构：

这个项目应该处于整体架构上的那一层：硬件 —— 服务器 —— 平台接口 —— 服务；

# 分析数据库的表（自己想要需要的核心表是哪些）

核心表：

根据表名和注释，找到数据库中核心的表，对核心业务影响最大的 3 至 5 个表；

# 分析项目目录（不要看代码，只看名字）

查看配置文件和项目依赖：

简略看下项目涉及的技术，回想下自己写过 demo（越简单越好），记不得去看博客笔记。

猜项目逻辑：

列出项目结构，根据文件夹（分析分层逻辑，内聚解耦思想的体现）和其下文件的文件名，猜出每一个模块的层级和这个模块的处理逻辑，做好标注。

根据之前分析的需求、表、项目分层逻辑、涉及技术:

用画图工具简单画出这个项目的运行流程。

# 模仿编写（看代码）

根据运行流程图，查看某一个业务的代码编写逻辑，进一步确定运行流程，并明确每一个层级之间的调用关系，在列出的项目目录具体标注功能和调用关系。

这个业务中涉及到不会的技术：通过搜索引擎搜索，这个技术一定要搜索具体的代码，而非泛泛的技术概念，对于技术的搜索答案，要选择最简单的描述即可。

列出查看业务 crud 等的代码调用层级和业务如何分层，画出图形，作为模仿的参照。

选择一个业务，根据已经分析的业务逻辑，编写一个业务逻辑的 crud 等简单功能。

# 写涉及技术 demo

根据项目依赖和配置文件:

去搜索对应技术的 demo，=== 不会或不明的技术。

搜索技术功能（答案要尽量简单）：

这个技术的功能是什么，解决的是什么问题，适合怎样的业务场景？一句话描述即可，不能多余一句话。

技术 demo（尽量简单，helloWorld 层级即可）：

搜索技术一个 demo，模仿编写：环境配置，项目依赖导入，项目的配置。这个技术功能（上步答案）的实现 demo，最简代码编写即可。

在项目中，去对应 demo 中的编写：

对于 demo 中未涉及的深入操作，根据官方文档，以及源代码，还有搜索引擎，去扩展编写即可。
